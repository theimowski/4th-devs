When this skill is selected, produce markdown notes using this structure:

1. Title
2. Short context paragraph (2-4 sentences)
3. Main sections with clear headings
4. Practical takeaways / next steps

Additional rules:

- Keep wording concrete and technical.
- Prefer short sections over large walls of text.
- If proposing architecture, include component responsibilities and interaction flow.
- If there are unknowns, state assumptions explicitly.
- When saving notes in the vault, preserve existing tone/style of neighboring notes.
